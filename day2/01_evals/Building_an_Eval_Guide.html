Tools

get_product

Looks up a product price from the catalog. Returns helpful error if not found.
Parameter
product: "jeans" | "t-shirt" | "sneakers" | …
Catalog (12 items)
jeans 49.99  shirt  29.99  dress  59.99
jacket 89.99 sneakers 74.99 hat  19.99
socks  9.99  hoodie 44.99 shorts 34.99
t-shirt 24.99 sweater 54.99 belt 24.99
On miss → returns "Product 'X' not found. Available products: belt, dress, …"

calculate

Performs arithmetic. The system prompt instructs Claude to never do mental math — always delegate here.
Parameters
op: "+" | "-" | "*" | "/" | "**"
input1: number
input2: number
Examples
# 3 shirts total
calculate("*", 29.99, 3) → 89.97

# 20% off jacket
calculate("*", 89.99, 0.80) → 71.992
Returns: a float (as string via execute_tool)
Why tool specs matter: The original specs had description: "get_product" and description: "calculator" — Claude had no idea what products existed or when to use calculate. After adding proper descriptions (listing all 12 products, adding an enum for operators), the multi-tool eval score jumped from 0% to 100%.

Graders

All graders share the same signature: grader(result, check, context) → {"score": 0.0 or 1.0, "reason": "..."}

response_contains Deterministic

String presence check

Checks if a substring appears anywhere in the final response text (case-insensitive).
check format
"49.99" # just a string
def grade_response_contains(result, check, context):
  text   = result["final_text"].lower()
  target = check.lower()
  if target in text:
      return {"score": 1.0, "reason": f"Found '{check}'"}
  return {"score": 0.0, "reason": f"'{check}' not found"}
PASS (1.0) — substring found
FAIL (0.0) — substring missing
response_numeric Deterministic

Numeric value check

Extracts all numbers from the response with regex and checks if any falls within ± tolerance of the expected value.
check format
{"value": 139.95, "tolerance": 0.10}
# Finds numbers like: 139.95, 1,234.56, -0.5
numbers = re.findall(r"-?[\d,]+\.?\d*", text)
for num in numbers:
  if abs(num - value) <= tolerance:
      return {"score": 1.0, ...}
PASS (1.0) — a number within tolerance found
FAIL (0.0) — no matching number in response
tool_use Deterministic

Tool call verification

Checks that the agent called a specific tool, optionally with specific arguments (partial match — only checks the keys you specify).
check format
{"tool_name": "get_product", "arguments": {"product": "jeans"}}
# Scans all tool calls in the transcript
for call in result["tool_calls"]:
  if call["name"] == tool_name:
    # partial-match specified args
    if args_match:
      return {"score": 1.0, ...}
PASS (1.0) — tool called (with matching args if specified)
FAIL (0.0) — tool never called / wrong args
llm_judge LLM-as-judge

AI-powered rubric check

Sends the query + response + a criterion to a judge model (Haiku). Parses PASS/FAIL from the first line. Used for open-ended questions where deterministic checks don't apply.
check format
"Response lists some of the available products"
# Judge prompt:
f"""Original query: {query}
Agent's response: {final_text}
Criterion: {check}"""

# Parse first line for PASS/FAIL
first_line = judge_text.split("\n")[0].upper()
if "PASS" in first_line: score = 1.0
PASS (1.0) — judge says PASS
FAIL (0.0) — judge says FAIL or unparseable
